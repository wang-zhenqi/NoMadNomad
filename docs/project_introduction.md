# NoMadNomad 项目立项说明书（Inception）

**文档类型**：立项 / Inception
**状态**：现行
**最后更新**：2026-03-12

---

## 1. 项目愿景与目标

### 1.1 愿景

NoMadNomad 是面向**技术型自由职业者**的 AI 驱动全流程项目管理工具，将 Upwork 从「读需求 → 写提案 → 拆任务 → 交付管理」的手动流程，转化为可复用、可追踪、成本可控的 AI 辅助工作流。

### 1.2 核心目标

- **效率**：提案撰写时间从约 30 分钟降至 5 分钟内。
- **质量**：通过结构化分析与模板化输出，提升方案专业性与一致性。
- **可追溯**：项目、需求分析、提案、任务与 Agent 运行均可落库与回溯。
- **实践**：作为敏捷开发、AI 应用与现代工程实践的完整案例。

### 1.3 目标用户

- 技术型自由职业者（初级至资深）
- 希望提升投标与项目管理效率的开发者
- 有意系统化记录与分享实践的技术博主

---

## 2. 问题与价值

| 问题 | 价值 |
|------|------|
| 写 Upwork 提案耗时长、重复劳动多 | 自动化分析 + 生成，缩短单次提案时间 |
| 技术方案与报价缺乏结构，难以复用 | 结构化分析与模板，便于迭代与复用 |
| 项目与提案散落，难以复盘 | 统一落库，支持版本与运行记录追溯 |
| AI 调用成本与稳定性不可控 | 契约化输出、校验、重试与成本度量 |

---

## 3. 范围边界

### 3.1 在范围内（MVP 及近期）

- Upwork 任务描述 → **需求分析**（结构化 JSON）→ **提案生成**（Markdown）→ 编辑、保存、版本查看。
- 项目 / 需求分析 / 提案 / Agent 运行记录的持久化与基础 API。
- 最简可用界面（或 API 先行 + 简单前端）完成上述闭环。
- 输出契约（Schema）、校验、重试与可观测（运行记录、耗时、错误）。

### 3.2 明确不在 MVP 范围

- 完整项目管理看板、甘特图、Trello/Jira 导出。
- Prometheus + Grafana 全量监控（MVP 可用基础指标/日志替代）。
- Upwork 自动抓取（MVP 以手动粘贴为主）。
- 向量检索 / 知识库深度能力（可后续迭代）。

---

## 4. 成功标准（MVP）

- **闭环**：固定样例输入下，稳定产出符合契约的需求分析 JSON 与提案 Markdown。
- **可回归**：关键路径有自动化测试（单元 + 集成），失败可追踪。
- **资源**：在约定内存约束（见下）下可稳定运行，不因单次请求导致 OOM。
- **可观测**：Agent 调用可记录模型、耗时、成功/失败，便于调优与排错。

## 5. 技术方向与约束

### 5.1 技术栈（与当前仓库一致）

- **后端**：FastAPI，异步；Python 3.11+。
- **前端**：Streamlit（快速迭代）。
- **AI**：LangChain + LangGraph（编排：分析 → 提案）。
- **数据**：SQLite（单机、低成本）；可选 sqlite-vec 用于后续检索。
- **质量**：pytest、black、isort、mypy（strict）、pre-commit。

### 5.2 架构分层（概念）

- **表现层**：Streamlit（项目/需求/提案的输入与展示）。
- **API 与编排层**：FastAPI 提供 REST；LangGraph 编排「分析 → 提案」工作流。
- **Agent 层**：需求分析 Agent、提案生成 Agent（后续可扩展润色、任务拆解等）。
- **数据层**：SQLite 存储项目、需求分析、提案、Agent 运行记录。
- **外部**：LLM API（如 OpenAI / DeepSeek / Ollama）。

### 5.3 内存约束（生产目标）

| 组件 | 内存分配 | 说明 |
|------|----------|------|
| LangGraph / Agent | 1.0 GB | 状态与上下文控制 |
| FastAPI | 256 MB | 异步与连接控制 |
| Streamlit | 512 MB | 缓存与分批加载 |
| SQLite | 128 MB | 缓存与 WAL |
| 系统与监控 | 其余 | 约 2 GB 总预算 |

---

## 6. 风险与缓解（摘要）

| 风险 | 缓解 |
|------|------|
| LLM 输出不稳定 | 结构化 Prompt、Schema 校验、重试与人工审核入口 |
| SQLite 并发与性能 | 单写者/串行写优先，批量提交，WAL，后续可读从库 |
| 2GB 内存压力 | 上下文与批次限制、依赖精简、监控与限流 |
| 开发时间与范围蔓延 | MVP 严格 P0 优先，迭代计划与 WIP 限制 |
| AI 成本失控 | 模型分层、缓存、配额与 token/成本记录 |

---

## 7. 项目阶段（路线图概要）

- **阶段 1（MVP，4–6 周）**：分析 + 提案闭环，数据层与 API，最简 UI，测试与可观测基线。
- **阶段 2（功能增强）**：润色与任务拆解 Agent，项目管理看板，监控与告警。
- **阶段 3（生产就绪）**：性能与安全加固，部署自动化，文档与备份策略。
- **阶段 4（扩展）**：多平台、协作与分享（持续）。
